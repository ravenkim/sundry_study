# 바텐더 — 화면 정의서

> 화면·라우팅 목록과 기능 매핑의 단일 원천은 [features.md](./features.md)입니다.

## 공통 레이아웃

| 영역 | 설명 |
|------|------|
| 상단 | 앱 이름 "바텐더" 또는 현재 화면 제목 |
| 본문 | 화면별 콘텐츠 |
| 하단 내비(권장) | 홈 / 대화 / 추천 / 냉장고 / 노트 |

모바일 퍼스트, Tailwind 기준 간격 및 타이포는 구현 단계에서 토큰화합니다.

## 1. 홈 — 바 카운터 (`/`)

**목표:** 진입 즉시 "바에 들어선 느낌"과 오늘의 한 마디.  
**연결 기능:** F-01, F-06(스트릭·등급 배지).

**구성 요소**

| 요소 | 동작 |
|------|------|
| 바텐더 인사 말풍선 | 진입 또는 새로고침 시 `GET /api/greeting` 로드 |
| 오늘의 추천 카드 | 술·안주·음악 프리뷀; 탭 시 `/recommend` 또는 상세로 이동(구현 선택) |
| 단골 스트릭/등급 | `GET /api/streak` 배지 표시 |

**UX 흐름**

1. 스플래시 후 홈 진입  
2. 인사 로딩 → 표시 → 카드 확인  
3. 하단 네비로 다른 탭 이동

---

## 2. 대화 (`/chat`)

**목표:** 바텐더가 먼저 말을 걸고, 대화를 통해 음료·안주·음악 추천까지 자연스럽게 연결.  
**연결 기능:** F-02.

**구성 요소**

| 요소 | 동작 |
|------|------|
| 바텐더 첫 메시지 | 화면 진입 시 자동 표시 — "어서오세요, 단골손님! 오늘 어떤 분위기세요?" |
| 메시지 리스트 | 사용자/바텐더 말풍선 구분 |
| 입력창 + 전송 | `POST /api/chat` 전송, 스트리밍은 구현 선택 |
| 안전 안내(선택) | 과음 언급 시 서버 플래그에 따른 부드러운 UI 힌트 |

**UX 흐름**

1. 화면 진입 → 바텐더 첫 질문 자동 표시 (로딩 없이 미리 정의된 문구 또는 `GET /api/greeting` 재활용)  
2. 사용자 답변 → `POST /api/chat` → 타이핑 인디케이터 → 응답 (음료·안주·음악 추천 포함 가능)  
3. 대화 이력은 세션 단위로 유지 (로컬 상태 기본)

---

## 3. 오늘 뭐 마실까 (`/recommend`)

**목표:** 기분에 맞는 술·안주·음악 세트 제안.  
**연결 기능:** F-03.

**구성 요소**

| 요소 | 동작 |
|------|------|
| 기분 선택 | 좋음/보통/우울/피곤/신남 등 세그먼트 또는 칩 |
| 선호 주종(선택) | 텍스트 또는 선택 UI |
| 결과 카드 | 술 상세, 안주 1~2, 음악 + YouTube 링크 버튼 |
| 다시 추천 | 동일 입력으로 재요청 가능 |

**UX 흐름**

1. 기분 선택 → (선택) 주종 → "추천 받기"  
2. `POST /api/recommend` 로딩  
3. 결과 카드 스크롤, 외부 링크는 새 탭/앱으로 열기

## 4. 냉장고 (`/fridge`)

**목표:** 남은 재료로 안주 결정, 부족 시 배달 아이디어.  
**연결 기능:** F-04.

**구성 요소**

| 요소 | 동작 |
|------|------|
| 재료 입력 | 태그 입력(칩 추가/삭제) |
| 추천 버튼 | `POST /api/fridge/recipe` |
| 레시피 카드 | 제목, 시간, 단계, 영양 한 줄 |
| 배달 제안 영역 | 재료 부족 시 안내 문구 표시 |

**UX 흐름**

1. 재료 나열 → 추천  
2. 레시피 모드면 단계별 펼침; 배달 모드면 구성 제안만 읽기 쉽게

## 5. 위스키 테이스팅 노트 (`/notes`)

**목표:** 위스키 기록·목록·취향 요약 루틴. 다른 주종도 기록 가능.  
**연결 기능:** F-05, F-05-3.

**구성 요소**

| 요소 | 동작 |
|------|------|
| 작성 폼 — 기본 | 술 이름(필수), 주종(필수), 별점 1~5(필수) |
| 작성 폼 — 테이스팅 | 향(Nose) / 맛(Palate) / 피니시(Finish) / 한 줄 감상 |
| 작성 폼 — 위스키 전용 | 주종이 위스키/버번이면 표시: 증류소 / 숙성연수 / 캐스크 종류 / 스타일·지역 |
| 작성 후 피드백 | 바텐더 코멘트 표시 |
| 노트 목록 | `GET /api/notes`, 주종 필터 (위스키 기본 선택) |
| 노트 카드 탭 | `GET /api/notes/:id` 상세 모달 또는 페이지 |
| 삭제 | 상세에서 `DELETE /api/notes/:id` |
| 취향 분석 | `GET /api/notes/analysis` — 위스키 기록 5개 미만이면 안내 문구 |

**UX 흐름**

1. 목록 진입 시 최신순  
2. 필터 변경 시 같은 API에 쿼리  
3. 작성 완료 후 목록 선반영 또는 재조회

## 6. 설정 / 루틴 (선택 경로 예: `/settings`)

**연결 기능:** F-06.

**구성 예시**

| 요소 | 동작 |
|------|------|
| 주간 리포트 카드 | `GET /api/report/weekly` |
| 알림 옵션 | OS 권한·로컬 설정(구현 범위에 따름) |
